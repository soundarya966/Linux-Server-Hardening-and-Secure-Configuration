# Linux Server Hardening & Secure Configuration

This project demonstrates basic Linux server hardening techniques using Ubuntu/Kali Linux.

## Objectives
- Secure SSH configuration
- Configure firewall
- Apply least privilege principle
- Disable unused services
- Enable automatic updates
- Monitor system logs

## Tools Used
- Kali Linux / Ubuntu
- UFW Firewall
- SSH
- Linux systemctl
- unattended-upgrades

## Steps Performed
1. Reviewed system users, ports, and services
2. Removed unused accounts
3. Disabled root SSH login
4. Configured SSH key authentication
5. Updated system packages
6. Enabled automatic updates
7. Configured firewall rules
8. Disabled unnecessary services
9. Secured file permissions
10. Reviewed authentication logs

## Outcome
Linux system successfully hardened against common security risks.
